# 本地量化交易系统（A股+港股，低频、低预算）规划

## Summary
- 目标：本地运行、数据驱动、ML/DeepLearning 产出信号，Open Claw 发送交易提示，人工确认执行；低频交易，覆盖 A股+港股股票/ETF。
- 成功标准：稳定的数据管线、可复现研究、可解释信号、可控风险与成本、可审计的决策记录。
- 约束：数据预算低（≤¥300/月），交易频率低，执行以人工确认优先，强调合规与数据版权。

## End-to-End Flow（端到端连接）
1. 需求与交易规则基线：确定交易宇宙（A股/港股、股票/ETF）、持仓规则、交易限制（如当日回转、最小交易单位等）与成本模型；把“人工确认”纳入执行流程与延迟假设。
2. 数据接入层：搭建统一采集接口与数据字典，输入包括持仓/交易流水、市场行情、基本面、公告、舆情/新闻。
3. 数据治理层：清洗、对齐交易日历、处理复权与公司行为、异常值检测、缺失填补；生成统一的“资产-时间”主键。
4. 特征工程：价格与成交量因子、基本面因子、事件/公告/新闻情绪特征、跨市场联动特征；构建可复现的特征版本。
5. 标签与训练集：定义“未来N天收益/回撤/胜率”等标签，做滚动切片与时间序列交叉验证，防止信息泄露。
6. 模型与信号：先基线（线性/GBDT）再逐步引入深度模型（序列模型/文本模型），输出可解释信号与置信度。
7. 组合与风控：从信号到仓位的映射（仓位上限、分散度、止损/止盈、换手约束），形成可落地的买卖清单。
8. Open Claw 提示与人工确认：生成可读“交易建议包”（理由、风险、预估收益/回撤、成本），发送消息并等待确认；记录确认与执行结果。
9. 交易后回放：追踪信号→执行→结果的闭环，更新特征与模型权重，形成持续迭代机制。

## Current Gap Register（按实施顺序）
1. 需求与交易规则基线
   - 当前状态：框架已定义交易配置，但主流程尚未使用持仓/交易流水来生成真实再平衡建议。
   - 关键缺口：缺少持仓驱动的 BUY/SELL/减仓逻辑，缺少审批状态机，港股最小交易单位仍是统一占位值。
   - 本轮优先动作：接入 positions/trades，按目标权重与当前持仓生成再平衡订单，并输出待审批清单。
2. 数据接入层
   - 当前状态：已支持 CSV、Baostock、Tushare、AKShare、RD-Agent 导入；港股官方接入仍为占位。
   - 关键缺口：港股自动拉取未落地，基本面/事件数据尚无统一抓取脚本。
   - 本轮优先动作：先稳定 A 股与本地 CSV 流程，保留港股为手工 CSV 接入。
3. 数据治理层
   - 当前状态：已有基础清洗、去重、复权因子处理。
   - 关键缺口：尚未把停牌、流动性、成交额门槛等规则真正应用到选股约束中。
   - 本轮优先动作：把 20 日平均成交额和可交易状态纳入约束。
4. 特征工程
   - 当前状态：已有价量、基本面、事件、跨市场、RD-Agent 特征。
   - 关键缺口：缺少与交易约束直接相关的流动性特征，缺少版本化元数据。
   - 本轮优先动作：补充 20 日平均成交额等交易约束特征。
5. 标签与训练集
   - 当前状态：已有 forward return 标签。
   - 关键缺口：训练仍是全样本拟合，没有 walk-forward/时间切分验证，存在信息泄露风险。
   - 本轮优先动作：实现 walk-forward 训练和样本外预测。
6. 模型与信号
   - 当前状态：已有 GBDT 基线和信号生成。
   - 关键缺口：预测目前以样本内拟合为主，信号输出不区分研究用历史信号与当下可执行信号。
   - 本轮优先动作：输出历史样本外信号，并从最新交易日生成可执行建议。
7. 组合与风控
   - 当前状态：已有 top-k 和等权配置。
   - 关键缺口：未落实最大换手、当前仓位约束、退出持仓逻辑和最小现金缓冲。
   - 本轮优先动作：加入基于持仓的目标仓位映射和最大换手限制。
8. Open Claw 提示与人工确认
   - 当前状态：可发送简短消息，但审批记录未接入主流程。
   - 关键缺口：没有结构化审批请求、审批状态更新和更可读的交易建议包。
   - 本轮优先动作：输出待审批订单文件，生成包含买卖摘要、风险说明和路径信息的消息。
9. 交易后回放
   - 当前状态：已有基础 backtest 脚本。
   - 关键缺口：延迟执行尚未真正进入收益计算，也没有将审批/执行结果回写闭环。
   - 本轮优先动作：修正 delay-aware backtest，并为审批记录预留回放输入。

## Data & Methods（数据与方法论）
- 市场与公告数据优先级：优先使用交易所/官方渠道与合规数据源；A股公告检索与ESG等官方数据入口可从上交所数据服务获取。
- A股低预算行情/基本面方案：
  - AKShare 作为开源数据聚合层，覆盖多类金融数据并提供统一接口，适合低预算接入与快速验证。
  - Baostock 提供免费中国股市历史数据接口，适合做日频/周频回测与原型验证。
  - Tushare 提供股票/基金/期货等行情与基本面数据，并提供API/SDK接口，可作为低预算的增强型数据源（需评估免费/付费权限与限制）。
- 港股历史数据方案：HKEX 提供官方历史数据产品与订阅平台；历史数据平台计划迁移至 HKEX Data Marketplace。
- 舆情/新闻数据（低预算思路）：以交易所公告与公开新闻为基础，先用“公告/新闻触发”的结构化特征与关键词情绪打底；后续再叠加更高成本的舆情数据源。
- 方法论参考（低频更匹配）：
  - 经典因子与价量结构：动量/反转、波动率收缩/扩张、成交量结构变化。
  - 事件驱动：公告、业绩预告、分红/回购、行业政策事件。
  - 机器学习：GBDT/随机森林作为主力基线；深度模型用于时间序列与文本情绪的增强。
  - 组合构建：以分散、低换手、成本敏感为核心，采用排名选股+风险约束的组合优化。

## System Challenges & Mitigations（难点与对策）
- 数据合规与版权：多源数据质量不一且许可复杂，优先官方/明示许可数据；避免违规分发与商业再利用。
- 数据质量与对齐：多市场交易日历差异、公司行为、停复牌、复权处理是核心难点；需要严格的数据版本与对齐逻辑。
- 过拟合与信号衰减：低频场景下样本量少，优先做“可解释的简单模型+稳健验证”，深度模型逐步引入。
- 交易成本与流动性：小资金更易被成本侵蚀，策略必须显式建模佣金、税费、滑点与最小成交单位。
- 人工确认带来的延迟：信号到执行有时间滞后，需要在回测中模拟“延迟执行”与“部分成交”的现实情况。

## Small Capital Strategy（小资金启动）
- 标的选择：优先流动性高的大盘股与ETF，减少冲击成本与滑点；谨慎参与小盘与冷门标的。
- 低换手与分散：降低频率、扩大持仓分散度，减少成本占比。
- 先实盘小仓测试：先以极小仓位验证信号稳定性，再逐步提高规模。

## Test Plan（验证与验收）
- 数据层：每日数据完整性检查、异常值/缺失值监控、对齐一致性抽样核验。
- 模型层：滚动窗口回测、时间序列交叉验证、避免未来函数与信息泄露。
- 交易层：含成本与延迟执行的仿真回测；人工确认流程是否闭环。
- 实盘层：纸面交易或小额实盘验证，连续运行至少 1-3 个月稳定后再扩大规模。

## Assumptions（默认前提）
- 持仓/流水数据以券商导出的 CSV 为默认来源，后续可升级为API对接。
- Open Claw 消息通道默认采用你现有可用的消息平台（如 Telegram/企业微信/Email 中任选其一），若需指定可调整。
- 风险控制参数默认保守（低换手、分散、严格成本约束），具体数值将在实盘前与你确认。
- 本规划仅用于量化研究与工程设计，不构成投资建议。
